Cloud Resume Challenge

Clouds are cool

Hello everyone, I am Jonathan. I have been working cloud adjacent for the last few years working in the cloud, but not designing it. I would like to know more so while I have some free time decided to take on the Cloud Resume Challenge.

Challenge

1. Certification

2 weeks study time | 2021

I received the AWS Cloud Practitioner certification in 2021. It is an introductory certification that orients you on the industry-leading AWS cloud.

2. and 3. HTML and CSS

4 hours | Feb 2023

Using a bootstrap template I went ahead and put my resume up in no time over the course of several evenings while watching TV. This was fairly easy as I have basic web design skills learned throughout the years thanks to Codeacademy

  • HTML
  • CSS
  • Jquery
  • Bootstrap

4. Static Website

1 hour | Feb 2023

I went ahead and created an S3 bucket named cloudresumewitcoski in us-east-1 (Shout out to Northern Virginia a few miles from my house). I set all features and encryption as default. I then uploaded the files of Step 1 and 2 into the S3 bucket. Reading various other people doing the challenge I needed to make sure the following were done:

  • Block public access — yes.
  • Server-side encryption — not required
  • Bucket versioning — not required.
  • Enable static website hosting

  • S3

5. HTTPS

1 hour | Feb 2023

Too see our website we then use CloudFront. Starting up a new CloudFront we set the S3 bucket from Step 3 to the origin. We then need to change the Origin access to Origin access control settings (recommended). So I went ahead and created the cloudfront distribution with the s3 bucket as the origin. I then copied the s3 bucket policy. After some time I was able to see my website!!!

  • Cloudfront
  • JSON

6. DNS

30 minutes | Feb 2023

I purchased the domain witcoskitech.com on Route 53. Then, I used AWS Certificate Manager to obtain an SSL certificate for the site, ensuring secure HTTPS connections.

  • Route 53
  • AWS Certificate Manager

7-8. Javascript and Database

2 days | March 2024

Implemented a visitor counter using JavaScript and set up a DynamoDB table to store the count. This involved writing a script to fetch and update the count, and configuring DynamoDB for on-demand pricing to keep costs minimal.

  • JavaScript
  • DynamoDB

9-10. API and Python

3 days | March 2024

Created a Lambda function in Python to interact with the DynamoDB table and set up an API Gateway to expose this function. Faced challenges with CORS configuration and Lambda function timeouts, which were resolved through troubleshooting and adjusting settings.

  • Python
  • AWS Lambda
  • API Gateway
  • boto3 library

11. Tests

1 day | March 2024

Wrote tests for the Python code to ensure reliability. Encountered and resolved issues with test imports and configurations.

  • pytest
  • Python testing best practices

12. Infrastructure as Code

2 days | March 2024

Used AWS SAM to define and deploy the backend resources, including the DynamoDB table, API Gateway, and Lambda function. This approach ensured consistent and repeatable deployments.

  • AWS SAM
  • YAML

13-14. Source Control and CI/CD (Back end)

2 days | March 2024

Set up a GitHub repository for version control of the backend code. Implemented a CI/CD pipeline using GitHub Actions to automatically run tests, package the SAM application, and deploy to AWS when changes are pushed to the repository.

  • Git
  • GitHub
  • GitHub Actions
